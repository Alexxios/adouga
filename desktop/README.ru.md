# Adouga — автоматическое определение игровой активности пользователя

Desktop-приложение на Python для мониторинга системы и AI-предсказания: делает скриншоты активного окна и классифицирует активность с помощью машинного обучения.

## Структура проекта

```
adouga/
├── backend/          # Backend-сервисы
├── common/           # Общие утилиты
├── desktop/          # Desktop-приложение
│   ├── src/
│   │   ├── main.py           # Точка входа пользовательского приложения
│   │   ├── dev_main.py       # Точка входа dev-приложения (сбор ML-данных)
│   │   ├── core/             # Общие модули (используются обоими приложениями)
│   │   │   ├── models.py         # Датакласс DataSample
│   │   │   ├── input_monitor.py  # Отслеживание клавиатуры и мыши
│   │   │   ├── hardware_monitor.py  # Снятие показаний CPU, RAM, GPU, диска
│   │   │   ├── window.py         # Определение активного окна
│   │   │   ├── screenshot.py     # Захват экрана
│   │   │   └── theme.py          # Константы темы UI
│   │   ├── app/              # Только пользовательское приложение
│   │   │   ├── inference.py      # Инференс ONNX-модели
│   │   │   └── ui/               # Страницы и виджеты UI
│   │   └── dev/              # Только dev-приложение
│   │       ├── recorder.py       # DataRecorder (скользящий буфер + экспорт)
│   │       ├── uploader.py       # Загрузка на YaDisk
│   │       ├── batch_uploader.py # Пакетная загрузка через очередь
│   │       ├── hotkeys.py        # Менеджер глобальных хоткеев
│   │       └── ui/               # Страницы dev-UI
│   ├── tests/
│   │   ├── core/             # Тесты общих модулей
│   │   ├── app/              # Тесты модулей пользовательского приложения
│   │   └── dev/              # Тесты модулей dev-приложения
│   └── pyproject.toml
└── ml/               # ML-модели и обучение
```

### Организация пакетов

- **`core/`** — общий фундамент, используемый обоими приложениями: мониторы железа/ввода, определение окна, захват экрана, модели данных и тема UI.
- **`app/`** — пользовательское приложение: ONNX-инференс и страницы UI (Live Stats, AI Analysis, Mouse Flicks).
- **`dev/`** — инструмент разработчика для сбора обучающих данных: recorder, YaDisk-загрузчик, пайплайн пакетной загрузки, хоткеи и dev-UI.

## Пользовательское приложение

Мониторинг системы в реальном времени и классификация активности на основе AI.

### Возможности

- **Live-мониторинг системы**: отслеживание CPU, RAM и активности ввода
- **Захват скриншотов**: автоматически снимает скриншоты активного окна
- **AI-классификация**: использует ONNX-модель для классификации активности (Gaming vs Not Gaming)
- **Мультиплатформенность**: работает на macOS и Windows
- **GPU-ускорение**: поддерживает CUDA, CoreML и другие execution-провайдеры

### Запуск

```bash
cd desktop
poetry install
poetry run python -m src.main
```

### Страницы UI

1. **AI Analysis**: скриншот с результатами AI-предсказания (класс, уверенность, провайдер)
2. **Live Stats**: графики CPU, RAM и активности ввода в реальном времени
3. **Mouse Flicks**: полярная диаграмма последних векторов движения мыши

### ONNX-модель

Бинарный классификатор на основе ResNet18, экспортированный в ONNX:
- **Вход**: RGB-изображения 224×224
- **Выход**: 2 класса (Not Gaming, Gaming)
- **Расположение**: `ml/models/model.onnx`

### Execution-провайдеры

Модуль инференса автоматически выбирает лучший доступный провайдер:
1. CUDA (NVIDIA) / TensorRT
2. CoreML (Apple Silicon)
3. DirectML (Windows)
4. ROCm (AMD)
5. CPU (fallback)

## Dev-приложение

Инструмент сбора обучающих данных для ML с автоматической загрузкой на YaDisk.

### Запуск

```bash
cd desktop

# Базовый запуск (сохранение локально, без загрузки)
poetry run python src/dev_main.py --no-upload

# С загрузкой на Яндекс.Диск (требуется .env с YADISK_TOKEN)
cp .env.example .env          # заполните токен
poetry run python src/dev_main.py

# Кастомные состояния и окно
poetry run python src/dev_main.py --states "Not Gaming" "Gaming" --window 300

# Все опции
poetry run python src/dev_main.py --help
```

### Возможности

- Интервал захвата 0.5 с (настраивается через `--interval`)
- Настраиваемая задержка перед первым захватом
- Поле имени тестировщика для параллельной работы нескольких тестеров
- Автоматическая пакетная загрузка на YaDisk (каждые 10 образцов)
- Ручной сброс через кнопку «Flush Now»
- Глобальные хоткеи (работают даже при отсутствии фокуса на окне)

### Глобальные хоткеи

| Хоткей | Действие |
|--------|--------|
| `Ctrl+Shift+R` | Включить/выключить запись |
| `Ctrl+Shift+N` | Перейти к следующему состоянию классификации |

## Разработка

### Зависимости

- **Мониторинг системы**: `psutil`, `pynput`, `mss`
- **UI**: `tkinter`, `matplotlib`, `pillow`
- **AI-инференс**: `numpy`, `onnxruntime` (CPU/GPU)
- **Поддержка macOS**: фреймворки `pyobjc` для определения окна

### Запуск тестов

```bash
cd desktop

# Все тесты
poetry run pytest

# Подробный вывод
poetry run pytest -v --tb=short

# Один модуль
poetry run pytest tests/dev/test_recorder.py

# Фильтр по имени
poetry run pytest -k "upload"
```

Тесты написаны в **чистом pytest-стиле** (плоские функции, без классов unittest).
Набор разделён на:
- **Unit-тесты** — весь внешний I/O замокан, работают мгновенно офлайн
- **Интеграционные тесты** — защищены `skipif`; требуют настоящих ресурсов ОС или переменных окружения

### Особенности платформ

#### macOS
- Требуются разрешения «Screen Recording» и «Input Monitoring»
- Для определения окна используются нативные PyObjC-фреймворки

#### Windows
- Определение окна через Win32 API
- Поддержка DirectML для GPU-ускорения
